# 송메이커 (Suno 같은 노래 만들기) — 개발 계획

주제·분위기·장르만 주면 **가사 + 반주 + (나중에) 보컬 + 커버 이미지 + 가사 싱크**까지 만드는 앱.
Suno와 달리 **밑바닥부터 학습하지 않고**, 잘 만들어진 오픈소스 조각을 **엮어서** 만든다.

## 확정한 방향 (사용자 결정)
- **첫 버전(MVP)은 보컬 없이**: 주제 → 가사 + 커버 + **뮤직랩 반주(악기 오디오)** + **가사 싱크 화면**.
- **내 PC·서버에 직접 (로컬·오픈소스 우선)**: 클라우드 유료 API 대신 로컬 모델/브라우저 내장.

## 큰 그림 — 누가 만드나
| 조각 | 방법 | 상태 |
|---|---|---|
| **곡 설계도**(조성·템포·섹션·코드) | 자체 규칙 엔진(SongCore) | ★ 1단계(지금) |
| **가사** | 로컬 LLM(Ollama) 훅 + 오프라인 초안 폴백 | 1단계 |
| **반주(악기 오디오)** | 뮤직랩 엔진: 코드→멜로디(Melody Maker)→드럼→MIDI→사운드폰트 | 2단계 |
| **가사 싱크(가라오케)** | MIDI 타임라인에 가사 음절 정렬 + 하이라이트 | 3단계 |
| **커버 이미지** | 분위기 기반 프로시저럴(오프라인) 또는 로컬 SD | 3단계 |
| **믹싱** | Web Audio로 스템 합치기·볼륨·페이드 | 3단계 |
| **보컬(노래 목소리)** | 로컬 오픈소스(ACE-Step/DiffSinger 등) | 4단계(가장 어려움) |

## 단계별 로드맵
- **1단계 — 곡 설계도 + 가사** (지금)
  - `SongCore.planSong({title,theme,mood,genre,length,seed})` → 섹션 구조·코드·템포·조성.
  - 가사: 로컬 LLM(Ollama `localhost:11434`) 호출 훅, 실패 시 오프라인 초안으로 폴백.
  - 단일 파일 `songmaker.html`, 유닛 테스트.
- **2단계 — 반주 오디오** ✅ (완료)
  - `Arranger.arrange(plan)`: 섹션 코드 → 멜로디(강박=코드톤·순차·에너지별 8/4분) RH + 쿵짝 베이스 LH → 멀티보이스 ABC.
  - `AbcMidi.abcToMidi`로 멜로디+베이스 MIDI → `MidiCore.makeBasicBeat`(장르별 드럼) → `mergeMidis`로 병합.
  - html-midi-player(GM 사운드폰트)로 재생 + .mid 저장 + 드럼 얹기/악보 연구소 핸드오프.
  - 유닛 6개(강박 코드톤·베이스 쿵짝·멀티트랙·드럼 병합·시드) + E2E(반주 생성·.mid 저장).
- **3단계 — 가사 싱크 + 커버** ✅ (완료)
  - `Stage3.buildKaraoke(plan, 가사블록)`: 섹션 시간(마디×박×60/템포)에 가사 줄을 배분 → {startSec,endSec} 타임라인.
    반복 코러스는 이름 매칭으로 가사 재사용. `lineAt(t)`로 현재 줄 찾기.
  - `Stage3.coverSpec(plan)`: 분위기별 팔레트 + 에너지 기반 도형 → 캔버스에 그라디언트 커버. PNG 저장.
  - 뮤직비디오 화면: 커버 위에 가라오케 자막(현재 줄 크게/다음 줄 흐리게), 재생 중 player.currentTime로 하이라이트.
  - 유닛 6개(타임라인 단조·총길이·lineAt·이름매칭·커버 시드) + E2E(커버 렌더·가사 표시·PNG 저장).
  - (믹싱은 4단계 이후로 미룸 — 현재 병합이 균형 잡혀 우선순위 낮음)
- **4단계 — 보컬** ✅ (완료 · 브릿지 방식)
  - 보컬은 외부 송 생성 AI로. `Stage3.buildVocalBrief(plan, 가사)` → [Style] 태그 + [Lyrics](영어 섹션 태그) 복사.
    → ACE-Step(로컬·GPU·무료) / ElevenLabs(월 ~11분) / Suno 에 붙여 완성 노래 생성.
  - 완성 노래(오디오) 가져오기 → `scaleKaraoke`로 타임라인을 오디오 길이에 맞춰 → 커버+가라오케 자막과 함께 재생.
  - 유닛 2개(buildVocalBrief·scaleKaraoke) + E2E(브리프 복사·오디오 가져오기).
  - 참고: 이 방식은 엔진이 자기 멜로디로 부름(우리 MIDI 멜로디와 다를 수 있음). 우리 멜로디를 그대로 부르는
    가창 합성(DiffSinger 등)은 한국어 지원·설치가 어려워 후순위.

## 원칙
- 뮤직랩 철학 유지: 가능한 한 오프라인·무료·테스트 가능·다른 랩과 연동(localStorage 핸드오프).
- 외부·로컬 모델은 **항상 graceful 폴백**(없어도 앱이 동작).
- 로컬 모델 연결부는 UI/설정에 두고, 순수 로직(설계·정렬)은 코어에 두어 테스트.
