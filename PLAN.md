# 피아노 ABC → 기타 타브(TAB) 변환기 — 개발 계획

## 목표
브로가드 뮤직 랩의 다른 도구들처럼 **단일 HTML 파일 로컬 앱**(`tab.html`)으로,
ABC 표기 피아노 악보를 붙여넣거나 파일로 열면 **기타 타브악보(ASCII TAB)** 로
변환해 보여주고 저장할 수 있게 한다. 핵심 변환은 **오프라인에서도 동작**하고,
abcjs(CDN)는 악보 미리보기 용도로만 사용한다(로드 실패 시 변환 기능은 그대로 동작).

## 파일 구성
| 파일 | 역할 |
|---|---|
| `tab.html` | 앱 본체. UI + 변환 엔진(TabCore)을 한 파일에 포함 |
| `tests/tab-core.test.mjs` | 변환 엔진 유닛 테스트 (Node 내장 test runner) |
| `tests/e2e.test.mjs` | Playwright E2E — 페이지 열기 → ABC 입력 → 변환 → TAB 출력 확인 |
| `index.html` | 메인 페이지에 타브 변환기 타일 추가 |

## 변환 엔진(TabCore) 파이프라인
1. **ABC 파싱** — 헤더(T/M/L/Q/K), 멀티보이스(`V:RH`, `[V:RH]` 프리픽스 — 이 랩의
   피아노 ABC 형식), 음표(임시표·옥타브·길이·점/브로큰리듬·셋잇단), 화음 `[CEG]`,
   쉼표 `z`, 마디선, 붙임줄, 코드 심볼 `"C"`, 장식음/슬러 무시
2. **음높이 계산** — 조표(장/단조 샵·플랫) + 마디 내 임시표 지속 규칙 적용 → MIDI 번호
3. **옥타브 자동 보정** — 피아노 음역을 기타 실음 음역(E2=MIDI 40 ~ 최고프렛)에
   최대한 넣는 12단위 시프트 자동 탐색(수동 조절 가능). 범위 밖 개별 음은
   옥타브 이동 후 경고 표시
4. **줄/프렛 배치** — 표준 튜닝 EADGBE(+카포). 단음은 손 포지션 추적 그리디
   (프렛 이동 최소화, 낮은 포지션·개방현 선호), 화음은 낮은 음부터 서로 다른 줄에
   스팬(≤4프렛) 제한 탐색으로 배치
5. **ASCII TAB 렌더링** — 6줄(e B G D A E), 음길이 비례 간격, 마디선,
   코드 심볼 행, 마디 단위 자동 줄바꿈, 제목/조성/카포/시프트 메타 헤더

## UI (다른 도구들과 동일한 디자인 언어)
- ABC 입력: 텍스트 붙여넣기 / `.abc` 파일 열기 / 샘플 불러오기 → **실시간 변환**
- 옵션: 보이스 선택(자동=오른손 RH 우선), 카포(0~7), 옥타브(자동/수동),
  최대 프렛(12~19), 타브 폭
- 출력: 원본 악보 미리보기(abcjs, 온라인일 때), ASCII TAB, 경고 목록,
  복사 / `.txt` 저장
- `?autotest=1` 쿼리로 자동 변환(테스트용 훅)

## 쿵짝 베이스 반주 (VER.1.1 추가)
멜로디는 높은 줄에 그대로 두고, **코드 심볼의 베이스 음을 낮은 줄(E·A·D)에 강박마다** 얹어
멜로디와 함께 치는 알터네이팅 베이스("쿵짝쿵짝")를 생성한다.
- **코드 파싱**(`parseChordRoot`): 루트/완전5도/슬래시 코드 베이스(C/E)·감5도 인식
- **박자별 쿵 위치**(`bassOnsets`): 4/4 = 1·3박(루트→5도), 3/4 왈츠 = 1박, 2/4·6/8 = 두 펄스
- **타임라인 병합**(`buildBassLine`): 강박에 멜로디 음이 있으면 같은 칸에 겹치고(함께 튕김),
  긴 음표/쉼표 아래로 강박이 지나가면 그 지점을 분할해 **베이스 전용** 칸을 삽입(멜로디는 울림 유지)
- **운지 배치**(`assignBass`): 낮은 세 줄에서 개방현 최우선·멜로디 손 위치에서 가깝게 →
  손가락이 닿는 위치 보장. 멜로디가 쓰는 줄은 피함
- UI: "베이스 반주" 셀렉트(없음 / 쿵짝)

## 자동 코드 삽입 (VER.1.2 추가)
베이스 반주가 켜져 있는데 **코드 심볼이 하나도 없으면**, 조성 기준으로 각 마디에
어울리는 "무난한" 다이어토닉 코드를 자동으로 골라 넣고 그 베이스를 만든다.
- `parseKey`가 으뜸음 pc·장단조·플랫여부까지 계산
- `diatonicCandidates`: 장조 = I·IV·V·vi·ii, 단조 = i·iv·V(장3화음 도미넌트)·VI·VII·III
- `autoChords`: 마디별 멜로디 음(길이 가중) 히스토그램 → 코드 구성음 커버가 큰 코드 선택
  (동점은 I·IV·V 우대), 선택한 심볼을 마디 첫 음에 부여 → 타브 코드 행에 표시
- 헤더/metaLine에 "자동 코드(조성 C): C Dm C C" 형태로 안내

## ABC ↔ MIDI 변환기 (midi.html)
Magenta Studio 등 외부 AI 음악 도구(Interpolate = 두 멜로디 사이 채우기)와 주고받기 위한
양방향 변환 다리. 핵심 엔진 `AbcMidi`는 자체 구현이라 오프라인·테스트 가능(abcjs는 미리보기용).
- **ABC → MIDI**: 검증된 ABC 파서(조표·임시표·화음·붙임줄·멀티보이스) 재사용 → 표준 MIDI
  파일(SMF format 1, PPQ 480) 바이트 직접 생성. 트랙0=템포/박자 메타, 보이스별 트랙.
  붙임줄은 마디선을 건너뛰어 병합(MIDI엔 마디 없음)
- **MIDI → ABC**: SMF 파싱(러닝 스테이터스·메타·note on/off 매칭) → 16분음표 그리드
  퀀타이즈 → 동시 시작음=화음, 마디 경계 넘는 음은 붙임줄 분할 → ABC 출력
- UI: ABC 붙여넣기/.abc 열기 → .mid 저장, .mid 드래그&드롭 → ABC 복사/.abc 저장, abcjs 미리보기
- **악보 연구소로 보내기**: MIDI→ABC 변환 결과를 `localStorage(broguard_lab_abc)`에 담아
  새 창으로 lab.html 열기 → 변환한 악보를 바로 악보 연구소에서 듣고 편집(연구소가 로드 시 키를 소비)
- 미리보기는 4마디마다 줄바꿈해 악보가 4마디씩 접혀 보이게 함
- 왕복(round-trip) 테스트로 음정·옥타브·화음·리듬·마디·붙임줄·템포/박자 보존 검증

## Interpolate — 곡 조합·전환 (interpolate.html, BETA)
Magenta Studio의 Interpolate 아이디어를 **알고리즘으로(오프라인·테스트 가능)** 구현. abcjs는 미리보기용.
- **Morph**: 두 멜로디를 8분음표 격자로 펼쳐 같은 길이로 리샘플 → 각 스텝의 음높이를
  다이어토닉(C장조) 인덱스 공간에서 t비율로 섞음(한쪽만 음이면 t 기준으로 페이드) →
  A → 섞임 → B 단계별 멜로디 생성
- **Bridge**: A 끝 음 → B 첫 음을 성부진행으로 잇는 연결 마디 생성 + A+브릿지+B 이어붙이기
- 검증된 ABC 파서 재사용, 결과는 ABC로 출력해 ABC↔MIDI·악보 연구소와 연동
- **파일 불러오기**: 멜로디 A·B에 .abc 열기 버튼
- **미리듣기(음색 선택)**: 자동 / 악기(abcjs 사운드폰트·온라인) / 기본 톤(Web Audio
  오실레이터·오프라인). 자동은 온라인이면 악기음색, 실패 시 기본 톤으로 폴백
- **코드 진행 반영**: 입력 코드 진행(마디 순환)의 코드톤으로 강박(1·3박) 스냅
  (Morph는 중간 단계만, Bridge는 연결음에)
- **AI Morph 모드**(온라인): 구글 Magenta MusicVAE(mel_2bar)를 CDN에서 로드해
  브라우저 로컬에서 실행(무료). ABC↔NoteSequence 변환은 코어에 두어 테스트, 신경망
  호출부는 UI에 둠. **AI 실패(모델 로드 실패 등) 시 알고리즘 Morph로 자동 대체**해
  결과가 비지 않도록 함. Morph·Bridge·AI Morph는 각각 선택 가능한 독립 모드
- **AI Morph 길이 확장**: mel_2bar 모델은 한 번에 2마디만 다루므로, A·B를 2마디 청크로
  쪼개(`abcToNoteSequence`의 `barOffset`) 청크마다 보간하고, 같은 보간 단계끼리
  `concatNoteSequences`로 시간축 이어붙여 원곡 길이(최대 16마디)만큼 길게 출력
- 유닛 26개 + E2E 12개(코드·파일·오프라인 재생·AI 칩·청크 확장 포함)

## Melody Maker — 무난작곡기 (melody.html)
코드 진행 → 규칙(강박=코드톤·순차 우선·기승전결 단일 클라이맥스) 기반 멜로디 후보 생성.
- 모드: 기승전결(전체·8마디) / 기·승·전·결 구간 전용 블록 / 8·16분 해상도 / 시드 재현
- **기승전결 한번에**(`generateSong`): 입력 코드 진행을 반복해 기·승·전·결 각 16마디(총 64마디)를
  한 흐름으로 이어붙인 완성곡 하나를 생성. 기(낮고 성김)→승(상승)→전(클라이맥스·촘촘)→결(하강·으뜸음
  종지) 곡선을 섹션별 모드로 자동 형성, 섹션은 `% ── 기 ──` 주석으로 구분
- 각 결과에서 🥁 드럼 얹기(merge.html로 handoff) 연동
- 유닛 17개(완성곡 64마디·결 종지·시드 재현 포함) + E2E(기승전결 버튼 포함)

## 확장실 — 16마디를 32마디로 (lab11.html)
16마디 곡을 32마디로 확장. **악구(프레이즈) 단위 반복**: 원곡을 4악구(기·승·전·결)로 나눠
각 악구를 [원악구][원악구 변형] 순서로 이어붙임(예: 기 4마디 C G Am Em → 8마디 C G Am Em·C G Am Em').
- 예전의 "한 마디 2배(C C G G)" 방식을 폐기 — `mapBar`로 악구 통째 반복 매핑
- 반복(뒤) 절반은 `varyNotes`로 약박 음만 이웃음으로 살짝 흔들어 "약간의 변형"; 전(절정)은 +3도
- 옵션 순환: 강(원곡 유지)→중(변형 발전)→리듬(반복 절반만 리듬 변주)

## 오디오 → MIDI 변환기 (audio.html)
노래·연주·허밍(MP3·WAV·마이크 녹음)을 분석해 ABC 악보·MIDI로. 두 가지 방식 제공(사용자 선택).
- **오프라인(단선율·기본)**: 자체 구현 `AudioMidi` 코어 — Web Audio로 디코드→16kHz 모노 다운샘플→
  **YIN 음정추적**(차분함수·CMND·포물선 보간) 프레임 분석→중앙값 평활(옥타브 오류 제거)→
  같은 음 세그먼트(최소 길이 필터)→8분음표 그리드 양자화→4/4 ABC(마디 넘김 붙임줄·4마디 줄바꿈).
  허밍·휘파람·솔로 악기 등 **한 번에 한 음**에 정확. 인터넷 불필요·유닛 테스트 가능.
- **AI(폴리포닉·온라인)**: 구글이 아닌 **Spotify Basic Pitch**(오픈소스·무료·TF.js)를 CDN에서 로드해
  브라우저 로컬 실행. 화음까지 채보 → `notesPolyToAbc`로 겹치는 음을 `[CEG]` 화음으로 조립.
  **로드/실행 실패 시 오프라인 단선율로 자동 대체**(결과가 비지 않도록).
- 마이크 녹음(getUserMedia+MediaRecorder), 템포·최소 음 길이 조절, abcjs 미리보기·재생.
- 결과: ABC 복사 / .abc 저장 / **.mid 저장**(내장 AbcMidi) / **악보 연구소로**(localStorage broguard_lab_abc) /
  **드럼 얹기**(broguard_drum_abc→merge.html) 핸드오프.
- 유닛 10개(YIN 탐지·양자화·화음 조립·AbcMidi 연동) + E2E(WAV 디코드→채보→.mid→핸드오프→AI 폴백)

악보를 따라 실제로 건반을 치며 연습하는 도구(리듬게임형).
- 내장곡 10곡(작은별·비행기·나비야·환희의 송가·엘리제·캐논·미뉴에트·아이네 클라이네·자장가·꿈속의 고향)
  또는 **직접 입력/.abc 파일**(BKP·연구소·오디오→MIDI 결과 등)
- 연습 모드: **기다려주기**(맞을 때까지 대기) / 박자+기다려주기 / 박자(안 멈춤), BPM 조절, 카운트인
- **떨어지는 음표 레인**(오선지) + 악보 하이라이트 + 화면 건반(오른손 멜로디·왼손 화음 A S D F…)
- 옥타브 무시·연음·자동 코드·코드 표시·자동 반주(소리), 실시간 채보(→ 직접 입력으로 보내기)
- **다른 랩에서 곡 받기**: 로드 시 `localStorage(broguard_practice_abc)`를 읽어 직접 입력에 채움(핸드오프)
- E2E: 랩 홈·내장곡·parseSong 파싱·핸드오프·JS 오류 없음(abcjs 미로딩 환경 포함)

## 테스트 전략
- **유닛**: `tab.html`에서 `<script id="tab-core">` 블록을 추출해 Node에서 실행.
  파싱(멀티보이스·임시표·화음·길이), MIDI 변환, 조표, 옥타브 자동 시프트,
  프렛 범위 보장, 화음 줄 중복 없음, 마디 수 보존, ASCII 정렬 검증
- **E2E**: 로컬 HTTP 서버 + Playwright(Chromium)로 실제 브라우저에서
  샘플 입력 → TAB 출력 → 복사/저장 버튼 동작 확인

## MIDI 드럼 병합 (merge.html, BETA) — 단계별 개발
목적: 뮤직랩 멜로디 MIDI + 외부(Magenta 등) 드럼 MIDI를 하나로 병합해 어디서나 재생.
소리는 html-midi-player(구글 Magenta 팀·무료)의 GM 사운드폰트로 드럼까지 자연 재생(온라인).
- **1단계(완료)**: MIDI 1개 불러와 재생(html-midi-player) + 구조 분석.
  `MidiCore`(채널 인식 SMF 파서): 트랙별 채널·노트·드럼 감지(채널 10)·GM 드럼 이름·PPQ·템포·마디.
